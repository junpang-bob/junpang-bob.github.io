This is my frist blog by use English,so it maybe has many grammatical mistakes.Please point out.
I want to discuss some vue-component disgin problems.let’s go.
First，component classified into two categories：common compoents，privacy components；

Common Compoents

Ureally，I Identify common components on layout component easily.We can discuss the style of the project with the UI designer early in the project.Solt component is a good tool for layout component.

We can inert some components in slot to get a Uniform Style.
Of course ,some common components have interaction. So, How do we identify them.When I meet them second time ,I would consider encapsulating them.I think we should’t handle Business Logic . Common components should focus on Interactive Logic.Simply put,we should’t calling interface to change real data.SomeTimes we meet some special common components ,they need calling interface to get data as config options.
I ureally use vueX actions
to async get data ,commit state. Because I don’t want to calling interface repetitive.

Privacy Components

About Privacy Components,I think we should more carefully,beaceuse common component come from privacy component.We can handle some simple interactive logic,we should’t use one component to handle many function,I think component just handle
interactive logic is best.If you want to call interface ,you can try to call it in top component.
That all,peace and love.
